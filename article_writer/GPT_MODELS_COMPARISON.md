## 📝 사용 권장 사항

### GPT-5-nano
- ✅ **추천 용도**: 빠른 요약, 분류, 간단한 분석
- ✅ **장점**: 가장 빠르고 저렴
- ⚠️ **주의사항**: 
  - `max_completion_tokens` 사용 (기본값 16000 권장)
  - `developer` 역할 사용
  - `temperature` 파라미터 제거

### GPT-5-mini
- ✅ **추천 용도**: 명확하게 정의된 작업, 빠른 응답이 필요한 경우
- ✅ **장점**: 빠르고 비용 효율적

### GPT-5
- ✅ **추천 용도**: 코딩, 에이전트 작업, 복잡한 reasoning 필요
- ✅ **장점**: 높은 성능, reasoning 지원

### GPT-4o
- ✅ **추천 용도**: 일반적인 텍스트 생성, 이미지 분석, PDF 처리
- ✅ **장점**: 안정적, 다양한 입력 지원
- ✅ **특징**: PDF 파일 직접 지원

### GPT-4o-mini
- ✅ **추천 용도**: 빠르고 저렴한 일반 작업
- ✅ **장점**: 빠르고 저렴, PDF 지원

## 🎯 칼럼 생성 프로그램에서의 선택

| 용도 | 추천 모델 | 이유 |
|------|----------|------|
| **일반 칼럼 작성** | GPT-5-nano | 빠르고 저렴, 충분한 품질 |
| **고품질 칼럼** | GPT-5 | 더 나은 reasoning, 깊이 있는 분석 |
| **PDF/이미지 분석** | GPT-4o | PDF 직접 지원, 안정적 |
| **빠른 프로토타입** | GPT-5-nano | 가장 빠른 응답 |

## 📌 주요 발견 사항

1. **GPT-5 계열 모델**:
   - `max_completion_tokens` 사용 (max_tokens 아님)
   - `temperature` 파라미터 미지원 (기본값 1만 사용)
   - `developer` 역할 사용 (system 역할 아님)
   - 이미지 입력 지원 ✅

2. **GPT-4o 계열 모델**:
   - `max_tokens` 사용
   - `temperature` 파라미터 지원
   - `system` 역할 사용
   - PDF 파일 직접 지원 ✅

3. **공통 사항**:
   - 모든 모델이 텍스트 입력 지원
   - 이미지 입력 지원 (vision 기능)
   - Base64 인코딩으로 이미지 전송
